This setup requires a MacOS no older than 10.15.x

After getting access to your Levels email & Google workspace, the first thing you'll be required to do is set up 2FA. You can do this in your Google account settings. After you set up 2FA, it's a good idea to download your "backup codes" and keep them backed up in two or more places.

Next, you'll want to download these apps:

  • Slack → To join the Levels team Slack, you'll need an invite from a member.
  • Harvest → You'll need an invite link from a team member to create an account with Levels. If you prefer tracking activity on your phone, you can also download the mobile app.
  • Zeplin → Get invited by a team member and sign up with your Levels email to get workspace/developer access.
  • Webstorm → You can set this up and begin the 30-day trial until you receive a license key from a team member.
  • Discord → You can use this invite to join the Levels Discord.

You'll also need to install...

  • Git → First, check if you have Git installed. If you don't, you'll be prompted to install it. git --version
  • Node.js → "macOS Installer" will be straightforward.
  • npm → This is installed automatically with Node.js, but you can check your version and update anytime. npm -v
    npm install npm@latest -g

Through invites from team members, you'll also gain access to Levels:

  • Github
  • JIRA
  • CircleCI

If you don't have a Github account, you'll want to sign up for one using your Levels email. If you already have a Github account with another email address, you can add your Levels email to your account and later use custom routing to direct Levels activity notifications to your Levels email. Then...

  • Set up 2FA on your account if you haven't already; 2FA will be required when you join the Lvls Group organisation on Github. Once you've set it up, it's a good idea to download your "recovery codes" and keep them backed up in two or more places.
  • Create a personal access token if you haven't already, and give it all the permissions;
  • When cloning repositories from Lvls Group, you'll be prompted for your username and password. Because 2FA is enabled, you'll need to copy and paste your personal access token into the password field when prompted, as your Github account password won't work in this case.
  • Once you've successfully cloned a repository, you can launch Webstorm and use File » Open » and select your repository folder to view the project.

Workflow

Levels uses a standard workflow wherein you can utilise Git or Github Desktop. For simplicity's sake, this step by step breakdown will assume that you're using the command line and just a basic text editor:

  1. View your project on JIRA and assign yourself a to do issue from the active Sprint.
  2. If you haven't already, clone the project to your machine git clone https://github.com/lvlsgroup/repository-name.git
  3. Navigate to the project folder cd ~/some-project-folder/repository-name
  4. Create a new branch using the name of the JIRA issue (this is usually a combination of letters and numbers i.e. EF-22) git checkout -b "issuename two-or-three-word-summary-of-intended-changes"
  5. Make the code changes you need and save your files. Then, add all changed files to staging git add * or choose which files to add to staging git add "your-file-name"
  6. Commit your changes git commit -m "short description of what you did"
  7. Push your changes to your new branch git push origin your-branch-name
  8. View the project on Github and open a new pull request. It's a good idea to link to the JIRA issue in the pull request, maybe summarise what you changed or add any relevant notes for reviewers or QA.
  9. You can tag someone on your team for a code review; once the changes are approved, you can merge your branch and begin the cycle again for a new issue.
  10. If you're tagged to review a pull request, you can check out the changes locally using Github Desktop, Github CLI (requires Homebrew), or you can fetch the changes git fetch origin pull/id-number/head:branch-name
    git checkout branch-name
